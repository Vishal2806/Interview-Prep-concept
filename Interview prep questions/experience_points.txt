# 🔹 1. Real-time activity tracking (Employee Monitoring System)

## 🧠 Kya hai?

Real-time tracking = user ki activity (mouse, keyboard, active app, URL, etc.) ko **live capture + backend me send + store + process** karna.

## 🎯 Interview Answer:

> “We used WebSocket-based communication to capture and send user activity continuously to the backend, where it was processed and stored in MongoDB for real-time monitoring and reporting.”

## 🔥 Follow-up Questions:

### ❓ WebSocket vs HTTP?

* HTTP = request-response (slow for real-time)
* WebSocket = persistent connection (fast, real-time)

### ❓ Why MongoDB?

* Flexible schema (activity data dynamic hota hai)
* High write throughput

# 🔹 3. AWS S3 + Pre-signed URLs

## 🧠 Kya hai?

Direct backend se file upload nahi — instead:
👉 Backend ek **temporary secure URL** deta hai
👉 Frontend directly S3 me upload karta hai

## 🎯 Interview Answer:

> “We used AWS S3 pre-signed URLs to allow secure direct uploads from client to S3 without exposing credentials, reducing backend load and improving scalability.”

## 🔥 Follow-up:

### ❓ Pre-signed URL kya hota hai?

* Temporary URL (expiry hoti hai)
* Specific file upload/download ke liye

### ❓ Benefits?

* Secure ✅
* Scalable ✅
* Backend load kam ✅
